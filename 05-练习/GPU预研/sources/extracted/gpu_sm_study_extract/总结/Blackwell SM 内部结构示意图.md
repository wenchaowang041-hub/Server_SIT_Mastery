核心结构要点（对应示意图）
4 个独立 Warp 调度器并行调度不同 Warp，支持单周期双发射（ALU + 访存），深度分支栈降低发散开销。
计算核心分区
CUDA Core：通用标量 / 向量计算（FP32/FP64）
第五代 Tensor Core + 稀疏执行单元：
支持 FP4，硬件自动稀疏调度，跳过零值，提升有效算力。
独立 TMA 单元（关键革新）完全硬件化的全局 ↔ 共享内存搬运，计算与访存解耦、流水线重叠，不占用 CUDA Core。
存储层次
Shared Memory ← TMA ← HBM3e
巨型片上 L2（不在 SM 内，在 GPU 级）大幅减少停顿
超大寄存器文件允许更高寄存器 / 线程，提升 ILP，不必一味追求超高占用率。
