NVIDIA Blackwell SM 内部结构与线程调度权威文档
版本：2026 年 3 月・正式版适用架构：Blackwell (B200 / GB200 / RTX 50 系列)核心主题：SM 内部结构、Warp 调度、TMA、稀疏 Tensor Core、执行模型
1. 文档概述
Blackwell 是 NVIDIA 2024–2026 年数据中心与消费级 GPU 的主力架构，其 SM（流多处理器）从传统计算集群升级为 微型自动化数据工厂。
核心突破：
计算与内存访问彻底解耦
第五代 TMA 实现硬件异步数据搬运
Tensor Core 支持动态稀疏调度（零值自动跳过）
4 路 Warp 调度器 + 双发射机制提升指令吞吐量
巨型寄存器文件与 L2 缓存大幅降低延迟
本文档完整解析 Blackwell SM 内部模块、数据流、调度机制。

3. SM 模块详细解析
3.1 Warp 调度集群（调度大脑）
每个 SM 包含 4 个独立 Warp 调度器。
功能：
每个周期可发射 1 条指令 / Warp
支持 Dual-Issue 双发射（ALU + Load/Store 并行发射）
深度硬件分支栈，显著降低 SIMT 分支发散开销
智能选择 “无停顿 Warp” 优先执行
调度模型：SIMT（单指令多线程）
3.2 执行数据通路（计算核心）
3.2.1 CUDA Core 簇
FP32 / FP64 通用计算
ALU 算术逻辑
SFU 特殊函数单元
负责所有通用并行计算
3.2.2 第五代 Tensor Core + 稀疏执行单元
Blackwell 最关键创新：
原生支持：FP4 / FP8 / FP16 / TF32
硬件自动稀疏调度
权重含 0 → 直接跳过 → 不占用周期 → 有效算力翻倍
3.2.3 Load / Store 访存单元
独立访存硬件，不占用计算核心。
3.3 内存子系统（瓶颈突破）
3.3.1 共享内存（Shared Memory）
SM 内高速 SRAM，用于线程间通信与临时数据。
3.3.2 第五代 TMA（Tensor Memory Accelerator）
Blackwell 架构核心杀手功能
独立硬件单元
自动完成：Global Memory ↔ Shared Memory 异步拷贝
计算与访存完全重叠
程序员不再需要手动调度数据搬运
3.4 超大寄存器文件
容量大幅提升
支持更高寄存器 / 线程
提升 ILP（指令级并行）
允许在不牺牲占用率的情况下提升单线程性能
4. Blackwell 线程调度机制（核心）
4 个 Warp 调度器并行工作
单周期单指令发射，支持双发射
TMA 异步拷贝 → 计算核心永不等待内存
Tensor Core 自动跳过零 → 无效指令不发射
深度分支栈 → 嵌套分支开销大幅降低
调度器优先选择无停顿 Warp → 利用率最大化

6. 代码优化指南（2026 实战）
尽可能使用 TMA 自动加载
不要过度追求占用率，平衡寄存器更重要
混合 ALU + 访存指令，触发 Dual-Issue
结构化稀疏分支可显著提升 Tensor Core 效率
FP4 / FP8 低精度推理优先使用 Tensor Core
