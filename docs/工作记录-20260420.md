## 二、华为 SP686C RAID 卡 UcfgFault 问题

**问题**: 同事做 HDD 热插拔后硬盘出现 UcfgFault 状态

**结论**: UcfgFault = 空闲盘失效状态。使用 hiraidadm 工具将盘设置为 Unconfigured Good 即可恢复：
```
hiraidadm -c setpd -p <盘ID> -s ucfggood
```
如有 external config 则需导入参考文档: `SP686C RAID控制卡 用户指南 04.pdf`（第 320 页表 B-9）

> 注：M7 日志管理的是 Kunpeng SoC 内部 BMC 网络，与 PCIe 灵活网卡无直接关系，但 CPU2 侧整体硬件初始化异常值得关注。

### 4.3 根因结论

**光模块正常，网卡未启动。** 三层不通：
1. **PCIe 配置空间不通** — VendorID/DeviceID 均为 0xFFFF，设备未响应配置空间访问
2. **I2C 管理总线不通** — BMC 通过 I2C_5 访问网卡管理芯片 CDR5902L 持续 NACK/Timeout
3. **驱动/固件未加载** — FirmwareVersion、DriverName、ChipVendor 全部为空

网卡处于 "WaitSignal" 物理层状态，无法完成链路协商。

**建议排查方向**:
- BIOS 中 PCIe 槽位是否已启用、资源分配是否充足
- 网卡供电是否正常
- 服务器 OS 下 `lspci` 是否能正确识别设备
- 网卡固件是否需要升级
- 交叉验证替换网卡排除硬件故障
