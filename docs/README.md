# Docs 导航

`docs/` 是仓库里的正式知识区，和日常复盘区分开。这里的内容应当更稳定、更适合长期复用。

## 目录入口

### [manual/](manual)

适合存放：

- 服务器硬件整合测试专题手册
- 新项目落地流程
- 检查清单
- 标准方法论

### [100-day-plan/](100-day-plan)

适合存放：

- 学习训练计划
- Day 原始资料
- 已整理博客稿
- 配套学习材料

### [bmc-cases/](bmc-cases)

适合存放：

- BMC / Redfish / IPMI 正式案例
- 已形成方法论的告警分析
- 可反复复用的排障记录

## 使用建议

如果一份内容已经不只是“当天记录”，而是以后还会反复查、反复用，就优先整理进 `docs/`。
