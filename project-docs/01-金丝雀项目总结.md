- **加速卡**: Atlas 300I A2 推理卡、Atlas 300V2 4G 视频解析卡、昇腾 910B
- **测试阶段**: EVT (工程验证测试)
- **BMC 版本**: 11.02.01.12 (openUBMC)、11.02.01.04

## 3. 自动化测试工具

### 3.1 DC Cycle 自动重启测试

**位置**: `脚本/DC/`

**工作原理**:
```
--start → 写入 state.json → ipmitool power cycle
开机后 rc.local → dc_cycle.py --auto → 健康检查 → next+1 → 下次重启
```

**配置**:
- 总轮次: 100 次
- 开机等待: 90s
- 出错停止: 是 (stop_on_error=true)

**健康检查项**: CPU、内存、PCIe、以太网、硬盘、NVMe、IPMI、dmesg错误

### 3.2 Reboot 详细打印脚本

**位置**: `脚本/详细打印reboot脚本/reboot/`

**架构**:
```
main.py (主控) → 调用各模块:
  ├─ info_cpu.py      → CPU信息采集
  ├─ info_mem.py      → 内存信息采集
  ├─ info_pcie.py     → PCIe设备信息
  ├─ info_eth.py      → 网络接口信息
  ├─ info_hdd.py      → 硬盘信息
  ├─ info_nvme.py     → NVMe设备信息
  ├─ info_ipmi.py     → IPMI/SEL信息
  ├─ info_dmesg.py    → dmesg错误收集
  └─ reboot.py        → 重启控制逻辑
```

**核心机制**:
- 每轮重启后采集 12+ 种系统信息
- 与基准文件 (base_round) 做 diff 对比
- PASS: 无差异；FAIL: 记录详细差异
- 自动收集 dmesg 错误和 IPMI SEL 事件
- 文件归档到 result/ 目录

**快速模式**: FAST_MODE=True 缩短超时时间

### 3.3 压力测试脚本

**stress_all.sh** - 三合一压测 (12小时):
1. NPU压测: `ascend-dmi -p --dur 43200` (8卡功耗测试)
2. CPU/内存压测: `stress-ng --cpu 240 --vm 18 --vm-bytes 100G --io 16`
3. 磁盘压测: fio randrw 4K 对 sda-sdi 9块盘并行

**FIO 12H脚本**:
```bash
fio --name=mix_workload --filename=/dev/sda \
    --rw=randrw --rwmixread=70 --bs=4k \
    --size=50G --numjobs=8 --runtime=43200
```

**USB3.0_TEST.sh**: USB3.0接口测试

### 3.4 DC 循环验证脚本 (麒麟系统)

**位置**: `金丝雀麒麟系统/reboot/`
- Python 脚本支持 DC cycle 自动化
- 带快照保存功能 (snapshot_dc1~dc15)
