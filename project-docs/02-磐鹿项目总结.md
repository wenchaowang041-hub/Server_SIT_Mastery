# 磐鹿项目总结

> 基于华为 Kunpeng + 昇腾 NPU 的 AI 推理服务器测试项目，覆盖 RC模式/单EP 多种硬件配置

---

## 1. 项目概况

- **平台**: 华为 Kunpeng 服务器 + 昇腾 NPU (310P3 / 910B 系列)
- **测试模式**: RC模式 (70T核心板 / 176T核心板)、单EP模式
- **操作系统**: openEuler 22.03 LTS SP3 / Kylin
- **工具链**: Ascend-mindx-toolbox (6.0/7.0/7.3.0)、npu-smi

## 3. 自动化测试工具

### 3.1 DC Cycle 自动重启测试 (磐鹿版)

**配置文件**: `dc_config.py.txt`

```python
# 核心配置
WAIT_BETWEEN_DC = 360      # DC间隔6分钟
WAIT_BEFORE_REBOOT = 30    # 重启前等待30s
MAX_REBOOT_COUNT = 101     # 最大101次

# 健康检查项 (15项)
HEALTH_CHECK_COMMANDS = {
    "psu", "temperature", "fan", "bmc_info", "bmc_selftest",
    "sel_events", "lsblk", "blkid", "lspci", "ip_link",
    "free", "dmesg", "nvme_list", "mount", "df"
}

# 快照对比 (10个文件)
SNAPSHOT_FILES = [
    "lsblk.txt", "blkid.txt", "lspci.txt", "ip_addr.txt",
    "dmesg_errors.txt", "ipmi_sdr.txt", "ipmi_sel.txt",
    "uptime.txt", "df.txt", "mount.txt"
]
```

**检查规则**:
- PSU异常: fail/critical/non-recoverable/absent
- 风扇异常: absent/fail
- 温度异常: critical/non-recoverable/upper critical
- RAID异常: Dgrd/Pdgd/Offln/Degraded
- 网络必须: state UP

### 3.2 Auto Reboot 脚本

```bash
MAX_REBOOT=300        # 300次重启
计数器: /tmp/reboot_counter
日志: /var/log/auto_reboot.log
```

**功能**:
- 带进度条显示
- 计数器持久化 (中断后可恢复)
- 自动清理

### 3.3 重启日志对比脚本 (main-1.py)

**对比文件** (15种):
```
cpu_all.log, info_cpu.log, info_mem.log, memory_all.log,
info_pcie.log, pcie_details.log, pcie_all.log, info_hdd.log,
info_raid.log, raid_all.log, info_nvme.log, ipmi_sdr.log,
ipmi_lan.log, ipmi_fru.log, info_eth.log, info_ifconfig.log
```

**机制**: 每轮与基准文件 diff，记录 PASS/FAIL

## 6. 已知问题

1. **FIO写测试失败**: 4K随机写和顺序写IOPS为0，可能磁盘配置问题
2. **NPU状态异常**: 310P3 Health=Critical, Power=NA
3. **5G网口阻塞项**: 远程开关机/控制/固件升级均为阻塞状态
4. **万兆网口性能**: "万兆网口跑不上去" - 性能未达标
