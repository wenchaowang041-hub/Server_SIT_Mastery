#### 用例：测试工具链一键安装与自检

**测试名称**：工具链一键安装（TestKit Bootstrap）\
**目的**：把“环境准备”从人工步骤变成可重复、可审计的一次性动作\
**预期结果**：脚本执行结束后，基础工具均可直接运行；GPU/NPU/TPU/DPU相关工具至少能给出“已安装/未安装/缺少驱动/缺少权限”的明确状态\
**工具/前提**：root权限；服务器可访问软件源/内网镜像；时间同步已配置（NTP/chrony）

**步骤（无脑照抄）**：\
1. 创建目录并进入：

    sudo mkdir -p /opt/hwcert/{bin,conf,logs/{raw,parsed,evidence},reports/{draft,final}}
    cd /opt/hwcert/bin

2\. 创建脚本文件：`install_testkit.sh`\
\[建议插入截图：终端粘贴脚本完成后的文件列表 `ls -l`\] 3. 执行安装：

    sudo bash install_testkit.sh 2>&1 | tee /opt/hwcert/logs/raw/install_testkit.$(date +%F_%H%M%S).log

4\. 运行自检：

    bash /opt/hwcert/bin/verify_testkit.sh | tee /opt/hwcert/logs/raw/verify_testkit.$(date +%F_%H%M%S).log

**完整命令（脚本）**：

    #!/usr/bin/env bash
    set -euo pipefail

    LOG_DIR="/opt/hwcert/logs/raw"
    mkdir -p "${LOG_DIR}"

    detect_pm() {
      if command -v apt-get >/dev/null 2>&1; then echo "apt"
      elif command -v dnf >/dev/null 2>&1; then echo "dnf"
      elif command -v yum >/dev/null 2>&1; then echo "yum"
      else echo "unknown"; fi
    }

    install_pkgs() {
      local pm="$1"; shift
      local pkgs=("$@")
      case "$pm" in
        apt)
          export DEBIAN_FRONTEND=noninteractive
          apt-get update -y
          apt-get install -y "${pkgs[@]}"
          ;;
        dnf)
          dnf makecache -y
          dnf install -y "${pkgs[@]}"
          ;;
        yum)
          yum makecache -y
          yum install -y "${pkgs[@]}"
          ;;
        *)
          echo "Unsupported package manager." >&2
          exit 1
          ;;
      esac
    }

    PM="$(detect_pm)"
    echo "[INFO] Package manager: ${PM}"

    BASE_PKGS=(
      "curl" "wget" "git" "jq" "lsof" "pciutils" "usbutils" "dmidecode" "lshw"
      "ethtool" "iproute" "net-tools" "numactl" "hwloc" "rsync" "screen" "tmux"
      "smartmontools" "nvme-cli" "mdadm" "lvm2" "bc" "python3" "python3-pip"
    )

    # 压力与基准
    STRESS_PKGS=("stress-ng" "sysbench" "fio" "iperf3")

    # 传感器/IPMI
    SENSOR_PKGS=("ipmitool" "lm-sensors")

    # RDMA/RoCE常用（不同发行版包名可能略有差异）
    RDMA_PKGS=("rdma-core")

    install_pkgs "${PM}" "${BASE_PKGS[@]}"
    install_pkgs "${PM}" "${STRESS_PKGS[@]}"
    install_pkgs "${PM}" "${SENSOR_PKGS[@]}"
    install_pkgs "${PM}" "${RDMA_PKGS[@]}" || true

    # GPU/NPU/TPU/DPU工具：只做“存在性检测 + 提示”
    cat >/opt/hwcert/bin/verify_testkit.sh <<'EOF'
    #!/usr/bin/env bash
    set -euo pipefail

    echo "== Basic Tools =="
    for c in stress-ng fio sysbench ipmitool sensors nvme jq lspci dmidecode ethtool iperf3; do
      if command -v "$c" >/dev/null 2>&1; then
        echo "[OK] $c: $(command -v $c)"
      else
        echo "[MISS] $c"
      fi
    done

    echo
    echo "== NVIDIA GPU =="
    if command -v nvidia-smi >/dev/null 2>&1; then
      nvidia-smi -L || true
      echo "[INFO] dcgmi: $(command -v dcgmi 2>/dev/null || echo MISS)"
    else
      echo "[MISS] nvidia-smi (likely driver not installed)"
      echo "       Install NVIDIA driver + CUDA stack per your OEM guide; DCGM docs: https://docs.nvidia.com/datacenter/dcgm/latest/index.html"
    fi

    echo
    echo "== AMD GPU (ROCm / AMD SMI) =="
    if command -v amdsmi >/dev/null 2>&1; then
      amdsmi version || true
    elif command -v rocm-smi >/dev/null 2>&1; then
      rocm-smi --showproductname || true
    else
      echo "[MISS] amdsmi/rocm-smi (install ROCm; AMD SMI docs: https://rocm.docs.amd.com/projects/amdsmi/en/latest/)"
    fi

    echo
    echo "== Huawei Ascend NPU =="
    if command -v npu-smi >/dev/null 2>&1; then
      npu-smi info || true
    else
      echo "[MISS] npu-smi (Ascend driver not installed or PATH missing)"
      echo "       Ascend quick install reference includes npu-smi验证方式。"
    fi

    echo
    echo "== MLPerf Inference Repo =="
    if [ -d "/opt/hwcert/mlperf_inference/.git" ]; then
      echo "[OK] /opt/hwcert/mlperf_inference exists"
    else
      echo "[INFO] Optional: git clone https://github.com/mlcommons/inference /opt/hwcert/mlperf_inference"
    fi
    EOF
    chmod +x /opt/hwcert/bin/verify_testkit.sh

    echo "[DONE] install_testkit completed. Run: /opt/hwcert/bin/verify_testkit.sh"

**Pass/Fail标准**：\
- **Pass**：通用工具全部OK；加速器工具至少能给出明确状态；日志写入`/opt/hwcert/logs/raw/`。\
- **Fail**：基础工具缺失（fio/stress-ng/ipmitool等）；脚本无日志；执行中断且无可定位错误。

**常见问题排查**：\
- `apt/dnf/yum`失败：检查DNS/代理/内网源；`cat /etc/os-release`确认发行版；抓包`curl -I <repo>`。\

## 测试用例写法标准与全局判定基线

### 全局原则

1）**任何压测都必须可回放**：命令、参数、版本、输入数据集、拓扑、温度与功耗曲线都要落盘。\
2）**先“可用性”后“性能”**：先枚举、健康、错误计数为0，再谈跑分；否则跑分毫无意义。\
3）**统一证据链**：所有模块输出至少包含：原始输出（raw）+ 关键字段提取（parsed）+ 结论（report片段）。

### 全局日志与证据规范（强制）

- 原始输出统一落到：`/opt/hwcert/logs/raw/<module>.<case>.<timestamp>.log`
- 关键字段（建议JSON）落到：`/opt/hwcert/logs/parsed/<module>.<case>.<timestamp>.json`
- 截图占位说明落到：`/opt/hwcert/logs/evidence/<module>.<case>.md`（写清楚该截什么、在哪里截）

### 全局“红线”Fail条件（任何模块触发即判Fail）

- `dmesg`或`journalctl -k`出现**持续增长**的硬件错误：PCIe AER致命错误、机器检查（MCE）、NVMe介质错误爆发、GPU/NPU Xid/掉卡等（具体在对应模块给出定位命令）。
- 温度/功耗长期越界：出现热降频且无法恢复；或PSU告警；或风扇异常。
- 同一测试在相同配置下结果不可重复（波动极大且无环境变化解释）。

### 标准用例模板（后续所有用例都按此结构写）

你会看到每个用例都包含： - 测试名称 + 目的 + 预期结果 + 工具/前提\
- 步骤（带命令、预期输出描述、截图占位）\
- 完整命令（bash代码块）+ 逐参数解析\
- 输出字段表格（字段｜含义｜正常范围示例｜异常处理）\
- Pass/Fail标准 + 常见问题排查 + 监控命令

#### 用例：整机信息一键采集（Inventory Snapshot）

**测试名称**：整机信息采集（Inventory Snapshot）\
**目的**：一次性获取“可审计”的系统基线：硬件、内核、驱动、拓扑、BMC基础可达性\
**预期结果**：输出包含CPU/内存/PCIe/NIC/盘/固件版本摘要；内核日志无致命错误；形成可归档证据\
**工具/前提**：`dmidecode/lshw/lscpu/lspci/lsblk/nvme-cli/ipmitool/journalctl`已安装

**步骤**：\
1. 创建采集脚本：`/opt/hwcert/bin/collect_inventory.sh`\
2. 执行采集：

    sudo bash /opt/hwcert/bin/collect_inventory.sh

3\. 检查输出目录：\
\[建议插入截图：`tree /opt/hwcert/logs/raw | head`\]

**完整命令（脚本）**：

    #!/usr/bin/env bash
    set -euo pipefail
    TS="$(date +%F_%H%M%S)"
    OUT="/opt/hwcert/logs/raw/inventory.${TS}"
    mkdir -p "$OUT"

    echo "[INFO] Collecting to $OUT"

    # OS / Kernel
    cat /etc/os-release | tee "$OUT/os-release.txt"
    uname -a | tee "$OUT/uname.txt"

    # CPU / NUMA
    lscpu | tee "$OUT/lscpu.txt"
    numactl -H | tee "$OUT/numa.txt" || true

    # Memory / DMI
    sudo dmidecode -t system -t baseboard -t bios -t processor -t memory | tee "$OUT/dmidecode.txt"

    # PCIe topology
    lspci -nn | tee "$OUT/lspci-nn.txt"
    lspci -tv | tee "$OUT/lspci-tree.txt"
    for dev in $(lspci -D | awk '{print $1}'); do
      lspci -vvvs "$dev" > "$OUT/lspci-vvv-${dev//[:.]/_}.txt" || true
    done

    # Storage
    lsblk -o NAME,SIZE,TYPE,MODEL,SERIAL,ROTA,MOUNTPOINT,FSTYPE | tee "$OUT/lsblk.txt"
    nvme list | tee "$OUT/nvme-list.txt" || true
    nvme list-subsys | tee "$OUT/nvme-subsys.txt" || true
    smartctl --scan | tee "$OUT/smart-scan.txt" || true

    # Network
    ip -br link | tee "$OUT/ip-link.txt"
    ip -br addr | tee "$OUT/ip-addr.txt"
    for nic in $(ls /sys/class/net | grep -vE 'lo|docker|veth'); do
      ethtool "$nic" > "$OUT/ethtool-$nic.txt" 2>&1 || true
      ethtool -S "$nic" > "$OUT/ethtoolS-$nic.txt" 2>&1 || true
    done

    # BMC sensors (if IPMI accessible in-band)
    ipmitool mc info | tee "$OUT/ipmi-mc-info.txt" || true
    ipmitool sdr elist | tee "$OUT/ipmi-sdr.txt" || true

    # Kernel logs
    dmesg -T | tail -n 2000 | tee "$OUT/dmesg-tail.txt"
    journalctl -k --no-pager | tail -n 4000 | tee "$OUT/journalctl-k-tail.txt"

    echo "[DONE] Inventory snapshot: $OUT"

#### 用例：CPU稳定性压力（stress-ng + sysbench双重验证）

**测试名称**：CPU满载稳定性（Compute Burn）\
**目的**：在高负载下验证系统稳定性，捕获MCE、热降频、供电不稳等问题\
**预期结果**：无掉核/重启；`dmesg`无机器检查；温度/功耗在可控范围\
**工具/前提**：stress-ng、sysbench、ipmitool/sensors

**步骤**：\
1. 开始监控（另开终端）：

    watch -n 2 'sensors 2>/dev/null | sed -n "1,120p"; echo; ipmitool sdr elist 2>/dev/null | egrep -i "Temp|Pwr|Fan" | head -n 50'

2\. 运行stress-ng（建议先10分钟冒烟，再按交付要求拉长）：

    stress-ng --cpu 0 --cpu-method matrixprod --verify --metrics-brief --timeout 10m

3\. 运行sysbench作为第二把尺子：

    sysbench cpu --threads=$(nproc) --time=300 run

4\. 检查内核日志：

    dmesg -T | tail -n 200 | egrep -i "mce|machine check|error|fatal|pci"

**完整命令**：

    stress-ng --cpu 0 --cpu-method matrixprod --verify --metrics-brief --timeout 10m \
      2>&1 | tee /opt/hwcert/logs/raw/cpu.stressng.$(date +%F_%H%M%S).log

    sysbench cpu --threads="$(nproc)" --time=300 run \
      2>&1 | tee /opt/hwcert/logs/raw/cpu.sysbench.$(date +%F_%H%M%S).log

**输出字段表格（sysbench）**：\
\|字段\|含义\|正常范围示例\|异常处理\| \|---\|---\|---\|---\| \|events per second\|计算吞吐\|同配置多次波动小\|波动大：查温度降频/电源策略\| \|latency avg/max\|平均/最大延迟\|稳定且可解释\|max飙升：查中断风暴/频率策略\| \|threads\|线程数\|=nproc\|线程数不对：权限/容器限制\|
