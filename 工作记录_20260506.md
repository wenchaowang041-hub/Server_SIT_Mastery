# 天数智芯 GPU 服务器测试工作记录

**日期**: 2026-05-06  
**服务器**: Huawei ARM64 Server (Kunpeng 920)  
**GPU**: 8x 天数智芯 BI-V150 (16 Cores)  
**系统**: openEuler (ARM64)  
**驱动**: Iluvatar CoreX Driver v4.4.0  

---

## 1. 系统安装与环境准备
*   **问题**: Kylin V10 ISO 和 Ubuntu 22.04 在 KVM 挂载安装时，卡死在 `EFI stub: Exiting boot services`。
*   **解决**: 调整安装方式，最终成功安装 openEuler 系统。
*   **网络**: 解决安装后 IP 地址无法获取及驱动安装导致的网络掉线问题。

## 2. 驱动安装与依赖修复
*   **环境**: 安装 Docker 29.1.3 及 `corex:4.4.0-release-llm` 镜像。
*   **报错**: 驱动安装器报 `Error 33: Missing dependencies`。
*   **解决**: 补全编译环境依赖。
    ```bash
    apt install -y gcc make dkms linux-headers-$(uname -r)
    ```
*   **验证**: `ixsmi` 正常输出，识别到 16 个 GPU 核心 (ID 0-15)，显存正常。

## 3. 磁盘扩容
*   **背景**: 默认 root 仅 70G，home 7.2T，空间分配不合理。
*   **操作**: 将 home 从 7.2T 缩减至 6.2T，释放出的 1T 扩容至 root。
*   **结果**:
    *   `/` (root): 1.1T
    *   `/home`: 6.2T

## 4. 稳定性测试 (Burn-in Test)
*   **工具**: `gemm_perf` (GEMM 性能与压力测试工具)
*   **配置**:
    *   全卡并行测试 (0-15 号核心)
    *   参数: `--d 2 --i [0-15] --l 10800000` (目标 12 小时)
*   **运行情况 (19:50 - 19:58)**:
    *   **负载**: 16 个进程全部 100% 运行，无掉卡，无报错。
    *   **温度**: 满载 8 分钟后，温度爬升至 85-95°C 区间。
    *   **功耗**: 单卡约 210-278W，部分卡功耗显示 `N/A`（可能是传感器/驱动 Bug，但不影响运行）。
    *   **降频**: 温度突破 90°C 后触发热节流 (Thermal Throttling)，SM 频率自动从 1500MHz 降至 750MHz 保稳定。
*   **结论**: 硬件稳定性通过初步验证，散热系统压力较大，热保护机制工作正常。
