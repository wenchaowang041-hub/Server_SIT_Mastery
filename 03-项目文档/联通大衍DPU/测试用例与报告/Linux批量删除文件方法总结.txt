Linux批量删除文件方法总结
linux怎么批量删除文件
在Linux中批量删除文件有多种方法，以下是常用的几种方式：

1. 使用通配符
bash
# 删除当前目录下所有.txt文件
rm *.txt

# 删除所有.log文件
rm *.log

# 删除所有以temp开头的文件
rm temp*
2. 使用 find 命令
bash
# 查找并删除当前目录及子目录中的.tmp文件
find . -name "*.tmp" -type f -delete

# 删除7天前的.log文件（更安全的方式）
find /path/to/dir -name "*.log" -mtime +7 -exec rm {} \;

# 删除空文件
find . -type f -empty -delete

# 删除超过100MB的文件
find . -type f -size +100M -delete
3. 删除目录及其内容
bash
# 删除目录及所有内容（谨慎使用！）
rm -rf directory_name/

# 删除多个目录
rm -rf dir1/ dir2/ dir3/
4. 按时间删除
bash
# 删除24小时内修改过的文件
find . -type f -mtime 0 -delete

# 删除30天前的文件
find . -type f -mtime +30 -delete
5. 交互式删除
bash
# 询问确认后再删除每个文件
rm -i *.txt

# 批量删除前先查看会删除哪些文件
ls *.tmp      # 先查看
rm *.tmp      # 再删除
6. 使用 xargs
bash
# 查找并删除（适合大量文件）
find . -name "*.bak" -type f | xargs rm -f

# 或处理带空格的文件名
find . -name "*.tmp" -type f -print0 | xargs -0 rm -f
重要安全提示 ⚠️
永远小心使用 rm -rf，特别是在根目录或重要目录中

删除前先备份重要数据

先使用 ls 或 echo 测试：

bash
# 先查看哪些文件会被删除
echo rm *.tmp
# 或
ls *.tmp
使用 trash-cli 更安全（先安装）：

bash
# 安装
sudo apt install trash-cli    # Debian/Ubuntu
sudo yum install trash-cli    # RHEL/CentOS

# 使用（文件会进入回收站）
trash-put *.tmp
设置别名防止误删（添加到 ~/.bashrc）：

bash
alias rm='rm -i'      # 删除前询问
alias del='trash-put' # 使用回收站
选择哪种方法取决于你的具体需求，建议先从最安全的方式开始！
