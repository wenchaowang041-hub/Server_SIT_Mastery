# 日常沉淀区说明

这里用于保存每天的工作记录、学习笔记和现场案例草稿。

和 `docs/` 的区别是：

- `daily-work-学习总结/` 更偏当天输入和阶段性沉淀
- `docs/` 更偏长期稳定、适合复用的正式知识

## 内容分类

### 每日工作记录

统一命名：

- `YYYY-MM-DD-工作记录.md`

适合记录：

- 当天问题
- 分析过程
- 结论
- 后续动作

### Day 学习笔记

统一命名：

- `Day13-xxx.md`
- `Day14-xxx.md`

入口：

- [Day学习索引.md](Day学习索引.md)

### 案例草稿

统一命名：

- `案例-xxx.md`

入口：

- [案例索引.md](案例索引.md)

### 原始工作日志

目录：

- [logs-每天工作日志](logs-每天工作日志)

适合存放：

- 原始命令输出
- 现场日志摘录
- 还不适合直接放进正式手册的原始记录

## 使用建议

如果一份内容已经整理成熟、以后还会复用，就继续往 `docs/manual/`、`docs/bmc-cases/` 或 `docs/100-day-plan/` 提升，不要一直停留在日常区。
