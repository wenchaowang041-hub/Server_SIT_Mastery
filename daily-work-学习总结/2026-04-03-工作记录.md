## 今日问题

### 问题1：NVMe FIO 批量测试脚本误纳入系统盘

- 现象：早期脚本在 `--all --include-write` 模式下未正确识别 `/dev/mapper/openeuler-root` 对应的底层盘，导致系统盘 `nvme5n1` 被纳入测试目标。
- 影响：存在误对系统盘执行写压力的风险。
- 初步判断：根因是系统盘识别逻辑没有覆盖 `LVM -> 分区 -> NVMe 盘` 的反查路径。

### 问题2：`stress-ng` 把 `--io 0 --hdd 0` 解释为最大 worker

- 现象：脚本原意是禁用 IO/HDD 压力，但 `stress-ng` 实际调度为 `256 io, 256 hdd`。
- 影响：短测和 12 小时稳测的测试类型被放大，结果不再是单纯的 CPU/内存稳定性测试。
- 初步判断：`stress-ng` 对参数 `0` 的解释与 `stress` 不同，脚本拼参逻辑需要按 `>0` 时才追加参数。

### 问题3：cycle 脚本在服务器 Python 3.9 环境下类型注解不兼容

- 现象：`python3.9 cycle_manager.py --mode dc --health-only` 首次运行时报错，定位到 `Sequence[str] | str` 语法不兼容。
- 影响：`cycle_manager.py` 无法直接在服务器侧落地验证。
- 初步判断：本地脚本用了 Python 3.10+ 的联合类型写法，需要回退到 Python 3.9 兼容写法。
